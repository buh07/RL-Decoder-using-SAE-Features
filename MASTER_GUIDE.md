## ⚠️ IMPORTANT — Read Before Citing Any Results

**Phase 4, 5.1, 5.2, 5.3, and 6 results are INVALID** due to SAE training bugs present in the original runs:
- `SAEConfig` defaulted to `use_relu=False` → L1 sparsity never engaged → features are not sparse
- `normalize_decoder()` was never called → decoder columns grew unbounded
- Decorrelation loss formula was incorrect → loss could go negative

The claimed 31.7% sparsity, 0.995–1.000 transfer ratios, Phi-2 r=0.754, and all feature descriptions are artifacts of broken training. These phases need to be re-run.

**Phase 5.4 multilayer SAE checkpoints** (`phase5_results/multilayer_transfer_v2/`) were retrained with the bugs fixed and are valid for use.

## 🏗️ Project Architecture

### Three Layers

```
1. RESEARCH LAYER (What we're studying)
   └─ How language models structure reasoning through depth
   └─ Do different models use different strategies?
   
2. ANALYSIS LAYER (How we study it)
   └─ Train 98 SAEs across all layers
   └─ Measure feature transfer between layers
   └─ Visualize reasoning flow patterns
   
3. IMPLEMENTATION LAYER (Where code lives)
   ├─ phase5/ - Core implementations
   │  ├─ phase5_task4_capture_multi_layer.py - Capture activations
   │  ├─ phase5_task4_train_multilayer_saes.py - Train SAEs
   │  ├─ phase5_task4_reasoning_flow_analysis.py - Compute transfers
   │  └─ phase5_task4_visualize_reasoning_flow.py - Generate charts
   ├─ src/ - Core SAE library
   └─ docs/ - This documentation
```

### Data Flow

```
Language Models
    ↓
Activation Capture (98 layers, 4 models)
    ↓
Activation Files (906 MB, organized by layer)
    ↓
SAE Training (98 sparse autoencoders)
    ↓
SAE Checkpoints (8x expansion, 50% sparsity)
    ↓
Transfer Matrix Computation (2,500 layer pairs)
    ↓
Analysis Results (JSON + 13 visualizations)
    ↓
Interactive Reasoning Tracker (NEXT: To be built)
```

## 🔬 Key Findings

### Phase 1 (Valid)
- BFS, Stack Machine, Logic Puzzle SAEs: R²=0.967–0.999 across all expansions
- Confirmed SAE framework works on ground-truth synthetic systems

### Phase 3 (Valid, Feb 22 2026)
- **Equation tokens**: detectable at layer 6 (83–97% per-class accuracy)
- **Reasoning connectives**: not detectable at layer 6 (0% per-class)
- **Conclusion**: Multi-step reasoning is not linearly encoded in any single layer → motivates Phase 5.4 multi-layer approach

### Phase 5.4 (SAE checkpoints valid; transfer analysis pending re-run)
- 98 SAEs retrained with corrected training across all layers of 4 models
- Transfer matrix analysis needs to be re-run against v2 checkpoints
- Previous transfer quality numbers (GPT2: 0.428, etc.) were from broken SAEs — do not cite

### Phase 4/5.1/5.2/5.3 (INVALID — needs redo)
- All results based on SAEs with no L1 sparsity (use_relu=False bug)

## 🔍 Troubleshooting

**Q: Where are the visualizations?**
A: `phase5_results/multilayer_transfer/reasoning_flow/visualizations/` - 13 PNG files showing transfer patterns

**Q: Can I export the raw data?**
A: Yes, see `phase5_results/multilayer_transfer/reasoning_flow/reasoning_flow_analysis.json` (JSON format)

**Q: What do the transfer ratios mean?**
A: Higher ratio = better feature transfer from source layer to target layer. Ratio 1.0 = perfect transfer, <0.1 = specialized (doesn't generalize)

**Q: Which model should I focus on?**
A: For reasoning tracking: GPT2-medium (universal features, easier to track). For stage-based analysis: Phi-2 or Pythia (clear layer specialization)
